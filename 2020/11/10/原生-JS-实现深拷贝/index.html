面试题中经常出现的一个问题，如何使用原生 JS 来实现深拷贝；因为深拷贝如果不限定范围，其实需要判断的分支比较多，那么面试官实际上是想考察一个面试者的掌握程度

现在我们来从头看下要实现一个相对合格的深拷贝代码，需要从解决哪些问题

需求背景

  • 基础数据拷贝
  • 对象或者数组拷贝
  • 日期对象、函数、正则拷贝
  • 解决循环引用导致栈溢出问题

实现思路

  • 基础数据类型 —— 直接返回
  • 对象或者数据 —— 需要递归
  • 日期对象 —— new Date(obj.getTime())
  • 函数 —— new obj.constructor()
  • 正则对象 —— new RegExp(obj)

我们先看下完整代码：

const getType = (obj: any): string => {
  return Object.prototype.toString.call(obj);
};

/**
 *
 * @param obj 拷贝对象
 * @param cache 缓存
 */
const deepCopy = (obj: any, cache = new WeakMap()) => {
  // 基础数据类型，直接返回
  if (obj === null || typeof obj !== "object") return obj;
  // 日期对象
  if (getType(obj) === "[object Date]") return new Date(obj.getTime());
  // 函数
  if (getType(obj) === "[object Function]") return new obj.constructor();
  // 正则
  if (getType(obj) === "[object RegExp]") return new RegExp(obj);

  // TODO 更多类型实现判断

  // 拷贝结果对象可能是 Obj 或者 []
  const copy: any = Array.isArray(obj) ? [] : {};
  // 有缓存，从缓存取值
  if (cache.has(obj)) return cache.get(obj);
  // 为了防止循环引用，缓存当前的对象类型，并设置为 {} 或 []
  cache.set(obj, copy);
  for (let i in obj) {
    const cur = obj[i];
    if (typeof cur === "object") {
      copy[i] = deepCopy(cur, cache);
    } else {
      copy[i] = cur;
    }
  }

  return copy;
};
